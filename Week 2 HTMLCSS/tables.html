Forms

Allow users to interact with websites to enter date

Forms can be short (login, search) or long (advanced searched, more inputs and options)

Usually data is sent to a server, which handles and sends a response

Googles web server will retrieve search results and send them back to the browser, a shopping sites credit card info will try to validate a card, etc

form is a container element, usually defines contents as a form. Input is a versatile element commonly used in forms

basic form


in order to submit a form to a server you need a form action to send the data to and a method to get or post

the action is a url on a server that should run a server-side script (a small program) to handle the form submission

the method attribue is set to get or post, get to send requests to data from server such as a search query, and post is to send data to a server to both text and binary data such as images, any sensitive data needs to be encrypted such as passwords

Input attributes

type: text, submit, many others

submit: submit button with inputs value as a button text, sends form data to a url specified in the form actions attribute

name: required, single word, tells server what the data is. you need a name otherwise the input will be ignored, always include a name with an input

value: default value to submit, also used for a submit button text

placeholder:

required: boolean, inputs with this attribute must be given a value before the form can be submitted

checkbox & radio. select values, checkbox can select multiple radio can only select 1

range: slider for a range between min and max values

color: color picker

file: file upload. when you upload a file you need to add enctype="multipart/formdata" to the forms start tag

More form elements

  • fieldset: gives structure to contents of a form, group sets of form controls together
  • legend: gives structure to contents of a form, set a caption at the top of a fieldset
  • textarea: multi-line text input form control. control visible lines with rows attribute and width with cols attribute. always set name attribute, can set a placeholder but not value attribute; if you want default text, place between tags
  • select & option: used together to create a dropdown list. similar to ol and li working together. select element contains a set of options, and requires a name attribute. each option surrounds a dropdown text option, each option requres a value attribute
  • label: surround a forms control caption text in a label element. associate the label and form control by giving the label a for attribute and the form control an id attribute, then set the values to match. that way if the label is clicked or touched, the associated form control is focused. example label for first name paired with input type text id first name.
